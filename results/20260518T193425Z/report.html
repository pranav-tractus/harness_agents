What to check next

Based on these results, here are the highest-priority actions for stakeholders:

01
Deploy Gemini 2.5 Pro for high-stakes extraction
Use Gemini 2.5 Pro when accuracy is critical and latency is acceptable—its 91.7% field match rate reduces downstream correction costs.
02
Choose Sonnet 4-6 for volume pipelines
For high-throughput scenarios where sub-5-second response times matter, Sonnet 4-6 offers the best speed (4.9 sec) while keeping accuracy near 90%.
03
Investigate OpenAI 5.4 performance gap
OpenAI 5.4 posted the lowest field match rate (82.2%) and highest mismatch count (4.28 per run). Review prompt compatibility or consider alternative models.
04
Optimize few-shot prompts for OpenAI models
OpenAI 4.1 performed well (90%) but OpenAI 5.4 lagged significantly. Test refined few-shot examples to see if newer model behavior differs from expectations.
05
Benchmark on additional datasets
This run covered only the 'downloaded' dataset. Expand to other datasets to confirm whether model rankings hold across varied extraction scenarios.
06
Monitor mismatch standard deviation trends
The 2.03 mismatch standard deviation suggests moderate variability. Track this metric over time to catch regressions in extraction consistency.

How to read these numbers

SUCCESS RATE — Share of runs that finished without a harness or HTTP error. High success means the run was stable; it does not prove the answers matched the reference.

AVG ELAPSED (S) — Average wall time per run in that bucket. Useful for latency comparisons.

AVG MISMATCH / EXPECTED RUN — Average count of fields that differed from the golden JSON when a reference existed. Lower is better.

FIELD MATCH — Fraction of compared fields that matched the golden output across runs in that bucket. Higher is better.
